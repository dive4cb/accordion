Accessible Accordion Widget 2

Introduction

This is a variation on the accordion pattern that departs from the ARIA Authoring Practices 1.0 accordion recommendation.

Instead of a multi-selectable tabs interface, with all its associated ARIA roles and attributes, as well as the huge amount of Javascript for keyboard and focus control, this accordion only uses two ARIA attributes.

  • aria-controls is used to establish a relationship between the triggering elements (buttons) and the content they control
  • aria-expanded is used on the triggering elements (in this case buttons) to show whether the associated contet is expanded or collapsed.

The pattern consists of headings containing buttons that, when activated, toggle the display of the associated content.

The triggering elements have to be focusable and actionable. A button is most appropriate semantically, a link is also valid.

aria-expanded is used, along with a visual indicator, to note whether the associated content is visible (aria-expanded="true") or hidden (aria-expanded="false".

The aria-controls attribute is used to provide a programmatically determinable relationship between the trigger and the content it controls. Technically, this is not necessary, as the content always follows the trigger element in the content order.

The headings are used as quick navigational aids for screen reader users who wish to jump past sections of displayed content to the next triggering element.

Javascript is needed to display and hide sections of content in response to the activation of its trigger, as wel as updating the aria-expanded attribute on the trigger.

Why is this approach better?

For the author, it is much easier to implement. It requires a lot less Javascript.

For the user it is simpler. The content order is clear, screen readers do not jump in and out of forms/application mode (as it does in tabs). In fact, the multi-selectable tab implementation of an accordion is very difficult to use for a screen reader user, because the tab content is inline with the tab. See further discussion.
